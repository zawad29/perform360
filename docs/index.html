Security

Zero-knowledge
by design.

Your passphrase derives an encryption key via Argon2id. Feedback is encrypted with AES-256-GCM before it reaches the database. Even with server access, the data is unreadable without the passphrase.

Step 01

Passphrase

Admin sets a company passphrase during setup. Never stored, never transmitted.

Step 02

Key Derivation

Argon2id stretches the passphrase into a cryptographic master key. Memory-hard, brute-force resistant.

Step 03

Encrypt

AES-256-GCM encrypts all evaluation responses before they touch the database.

Step 04

Store

Ciphertext at rest on your server. No escrow, no backdoors. Full GDPR export on demand.

Get Started

One command.
You're live.

$ curl -fsSL https://raw.githubusercontent.com/tamzid958/perform360/master/docker-compose.yml -o docker-compose.yml
01

Create & Fetch

mkdir performs360 && cd performs360, then grab the compose file with the command above. Requires Docker with Compose and an external PostgreSQL database.

02

Configure

curl -fsSL …/.env.production.example -o .env, then edit it. Set DATABASE_URL, NEXTAUTH_SECRET, your public URL, and email provider.

03

Pull & Start

docker compose pull && docker compose up -d. App runs on port 3000. Visit /setup-encryption on first boot to initialize encryption keys.
